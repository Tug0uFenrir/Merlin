# 🧙 梅林计划：征集跑团日志

## ✨ 项目愿景

梅林（Merlin）是一个正在开发中的,专为桌面角色扮演游戏（TRPG）而生的大型语言模型。它不是一个通用的聊天AI，而是一个专业服务于TRPG玩家的AI模型。

我们的目标是让“梅林”能够：

- 📖 深度理解COC、DND等多体系规则书，进行精准的规则问答与结算。
- 🎭 稳定扮演从英勇的圣武士到疯狂的传奇调查员等任何角色，保持人格的高度一致。
- 🧩 情境推理在复杂的跑团叙事中，理解剧情脉络、角色关系与线索网络。
- 🤖 无缝集成作为未来智能骰娘的“大脑”，提供真正的智能交互。

这一切的基石，是您所创造与参与的、充满智慧与故事的跑团记录。

## 📜 我们需要什么样的日志？

任何真实的、有趣的、及具深度地跑团记录都将会是宝贵的宝藏。我们尤其需要：

### 🏆 理想日志的特征：

- **完整性**：包含从开场、探索、交涉、战斗到结团的相对完整的故事线。
- **互动性**：有丰富的玩家与主持人（KP/GM）之间的对话、决策和角色扮演。
- **规范性**：包含游戏中进行的技能检定、战斗轮、理智鉴定等规则交互过程。
- **多样性**：欢迎各种规则（COC 7th, DND 5e, 最终物语，三角机构，迷雾之城等其他的，或是原创的规则）、风格（硬核、搞笑、戏剧）和平台（文字团、语音转文字）的日志。

### 🔧 格式要求：

- **语言**：以中文为主的日志。
- **格式**：.txt, .docx, .md 或论坛导出文本均可。
- **内容结构**：需要清晰区分说话人（如 [KP], [玩家A] ）和旁白/描述。

## 🚀 如何贡献您的日志？

您可以通过以下任一方式，安全、便捷地贡献日志：

### 方式一：GitHub提交（推荐）

1. Fork 此仓库。
2. 将您的日志文件（请移除任何真实个人信息）放入 `logs/` 目录。您可以创建子文件夹，如 `logs/coc/` 或 `logs/dnd/`。
3. 提交 Pull Request。我们将在合并前进行审核。

### 方式二：邮件发送

如果您不熟悉GitHub，可以直接将日志发送至我们的项目邮箱：alang457@163.com

- **邮件主题**：[梅林日志贡献] - 规则 - 简短描述
- **邮件正文**：请注明您希望使用的署名方式（如社区ID、匿名），以及模组名称（如果涉及知名模组）。

### 方式三：使用我们的预处理工具

我们正在开发一个本地化的日志清洗工具，它将自动帮您：

- 脱敏（去除社交账号、真实姓名等）。
- 标准化格式。
- 生成贡献元数据。

工具发布后，将更新于此仓库。

## 🛡️ 隐私、版权与伦理承诺

我们深知您的跑团记录是珍贵的故事与回忆。我们郑重承诺：

- **完全脱敏**：所有被采纳的日志，都会彻底移除其中的真实姓名、联系方式、地理位置等个人的可识别信息。
- **版权尊重**：
  - 您保留原始日志的一切版权。
  - 您授予“梅林计划”仅为模型研究、训练与测试目的使用该日志文本的权利。
  - 您的贡献不会用于任何商业产品或直接盈利。
- **可控公开**：训练完成后，用于训练的数据集（经多重脱敏与混淆处理）将以严格的研究用途有限开放。
- **署名选择**：您可以选择以您的社区ID署名，或完全匿名。

## 🏗️ 数据处理流程

您的日志将经过以下步骤，最终成为“梅林”的知识：

```
原始日志 -> 自动脱敏 -> 格式标准化 -> 质量审核（人工抽样）-> 分类与打标 -> 进入训练数据集
```

## ❓ 常见问题

**Q：我的团里有“黑历史”或者特别中二的情节，会不会很尴尬？**  
A：完全不会！正是这些充满真情实感、或严肃或滑稽的互动，才是AI学习“人性”与“扮演”的最好教材。所有数据都将被严格脱敏并混合处理。

**Q：语音团的录音可以吗？**  
A：可以，但我们目前会优先处理文字记录。如果您有语音转文字后的文稿，同样欢迎。

**Q：如何确认我的日志被采用了？**  
A：我们会在合并Pull Request或邮件回复时确认。同时，我们会定期在仓库的 `ACKNOWLEDGEMENTS.md` 文件中更新贡献者列表。

**Q：这个项目是开源的，模型会开源吗？**  
A：是的，这是我们的核心承诺之一。“梅林”基座模型将在非商业许可下开源，以供社区研究、改进和创造。
